## 9. ユーザーフィードバック (User Feedback)

### 9-1. Anchor Settings の冗長性

> 「Anchor Settings (基準日設定)」は「Phase Steps (工程一覧)」で設定できる要素ではないか？

**検証結果**: ✅ 正しい

Anchor Settings で行う操作：
- Anchor Phase 選択（どの工程を基準にするか）
- Date 入力（基準日）
- 開始/完了 選択

これらは **Phase Steps 内で既に設定可能**（📌マーク表示、行内日付設定）。

| 問題 | 改善案 |
|:---|:---|
| Timeline Configuration セクションが冗長 | Phase Steps に統合し、セクション削除 |
| 2箇所で同じ設定ができる混乱 | 単一の操作ポイントに集約 |

### 9-2. デザイン方向性の見直し

> マージンが大きく、ビジネス感が不足している

**検証結果**: ✅ 妥当な指摘

| 現状 | 問題点 | 改善案 |
|:---|:---|:---|
| グラスモーフィズム | ポートフォリオ向けの「見せるUI」 | フラットでクリーンなデザイン |
| 大きなマージン (`2rem`) | 情報密度低、スクロール量増加 | コンパクト化 (`1rem`) |
| 絵文字アイコン | カジュアルすぎる | SVGアイコン or テキストのみ |
| ネオンブルーアクセント | 「ゲーム/テック感」が強い | 落ち着いたブルー/グレー系 |

**参考デザイン**: Notion / Linear / Jira のようなミニマルで高密度なビジネスUI
